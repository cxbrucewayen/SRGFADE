# SRGFADE release-readiness audit

Audit date: 2026-07-12

## 1. Scope

This audit covers the complete `SRGFADE-code` release tree, the wheel, and
the source distribution. The paper and the frozen authoritative
implementation were read only. The release tree is limited to package
metadata, source, documentation, citation and licence files, one example,
tests, and release validation records.

The accepted repository files are `.gitignore`, `BASELINE.md`,
`CITATION.cff`, `LICENSE`, `MANIFEST.in`, `README.md`, `pyproject.toml`,
`requirements.txt`, the two release reports, the files below `examples`, the
files below `srgfade`, and the files below `tests`. No other file was present
after cleanup.

## 3. Findings, risks, and fixes

### Finding 1: historical signature text in regression code

Fact: two calls in `tests/test_regression.py` named the frozen-source
perturbation argument directly.

Risk: publishing that name would reintroduce terminology that is not used by
the manuscript or the public API.

Fix: the regression adapter now compares the public and frozen constructor
signatures and maps the single differing coefficient by introspection. The
same default and nondefault numeric comparisons remain in place. The complete
test suite passed after the change.

### Finding 2: incomplete source distribution

Fact: the initial source distribution contained the package and tests but
omitted `CITATION.cff`, `requirements.txt`, and `examples/quick_start.py`.

Risk: a reader installing from the source archive would not receive files
referenced by the README.

Fix: `MANIFEST.in` now includes the citation file, licence, README,
requirements, example files, and test files. The rebuilt source distribution
contains all required members.

### Finding 3: deprecated licence metadata form

Fact: the initial build completed with a setuptools deprecation warning for
the table-form licence field.

Risk: the form has a published removal deadline and weakens future build
compatibility.

Fix: `pyproject.toml` now uses the licence expression
`LicenseRef-SRGFADE-Source-Available`, explicitly includes `LICENSE`, and
requires setuptools 77 or newer. The final wheel and source distribution
built without warnings. The wheel metadata records the expression and the
licence file, the wheel contains the licence under distribution metadata, and
the source distribution contains `LICENSE` at its root.

### Finding 4: population-size contract mismatch

Fact: the README allowed any positive population size, while a population of
one could not supply the donor set required by the differential operator.

Risk: an input described as valid could fail inside the search loop.

Fix: the public base class now requires a population of at least four, the
README states the same boundary, and tests cover values zero through three.
Valid fixed-seed regression cases remain numerically equal to the frozen
implementation.

64-bit. The isolated build environment used Python 3.11.5 and build 1.5.1.
Its isolated backend environments used setuptools 83.0.0 and wheel 0.47.0.

Two newly created Python 3.11 environments were used outside the release

start with 600 counted evaluations. The second environment installed the
source distribution with its test extra, imported the installed package from

The source-tree verification also passed all 28 tests under Python 3.11.5.
The wheel test was executed from a separate working directory with a separate
pytest root and importlib import mode, which prevented the repository source
from shadowing the installed package.

## 5. Terminology and attribution check

The README and runtime source use the four manuscript-aligned mechanisms:

- subspace rotation crossover
- gradient-field-guided mutation
- annealed Gaussian perturbation
- restricted simulated-annealing acceptance

No fifth mechanism or internal mechanism number is present. The names in
`CITATION.cff` and `pyproject.toml` are the intended manuscript attribution.
No email address, account identifier, contact field, credential, or private
key is present.

The frozen optimizer and shared-base SHA-256 values remain
